# Canonical docs for Punk workflow

Start here for meaningful repo work:

1. `work/STATUS.md` — current focus, `selected_next`, blockers, recent evidence.
2. `work/goals/<selected>.md` — bounded scope, acceptance, exclusions.
3. `docs/product/START-HERE.md` — active posture and current trust surfaces.

Load more only if the selected goal needs them:

- `docs/product/ROADMAP.md` — phase ordering and promotion gates.
- `docs/product/PROJECT-MEMORY.md` — manual Work Ledger boundary and repo-tracked memory rules.
- `docs/product/DOGFOODING.md` — Level 0 closure semantics and what is not yet self-execution.
- `docs/product/CRATE-STATUS.md` — active-core crate boundaries.
- `docs/product/FLOW.md` — flow semantics.
- `docs/product/TELEMETRY.md` — event and inspectability expectations.
- `docs/product/DOCUMENTATION-MAP.md` — canonical owner for docs.

If a truth is already owned by a canonical doc, link to it or update it there instead of duplicating it in the skill.
