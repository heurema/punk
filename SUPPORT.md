# Support

## Where to ask for help

Use GitHub Issues for:

- bug reports;
- bounded feature requests;
- bounded questions about current repo behavior or current active surfaces.

## Before opening an issue

Please check:

- `README.md`
- `AGENTS.md`
- `work/STATUS.md`
- `docs/product/START-HERE.md`
- `docs/product/DOCUMENTATION-MAP.md`
- `CONTRIBUTING.md`

## When not to use public issues

Do **not** use public issues for:

- vulnerability details;
- secrets, credentials, or private infrastructure data;
- anything that requires a private security handoff.

See `SECURITY.md` for that path.

## Response expectations

`punk` is an experimental public build with finite maintainer capacity.
Clear scope, exact file paths, and reproducible steps make reports much easier
to handle.
