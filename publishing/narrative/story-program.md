# Story program

## Purpose

This file is the manual editorial map for the current public narrative.

It turns the Journal into a small set of active story arcs that can be published manually now and later adopted by a future pub module.

## Rule

Keep at most 3 active story arcs at once.

Everything else stays queued.

## Active story arcs

### 1. Origin: why Punk exists

Status: active

Primary story:

- `publishing/stories/story-0001-origin.md`

Journal sources:

- the trust question in the early eras
- the shift from reading code to building trust through process
- the line from raw agents to contracts, gates, and proof

Core angle:

- `punk` exists because the trust problem became explicit

Suggested first posts:

- first public build note
- the question that started Punk: can you stop reading code and still trust the result?

### 2. Core-first: boundaries before behavior

Status: active

Primary story:

- `publishing/stories/story-0002-core-first.md`

Journal sources:

- convergence from adjacent systems into one runtime
- the reset into a smaller, more disciplined core
- current Punk rule: activate behavior slowly

Core angle:

- the first build is not feature breadth; it is boundary discipline

Suggested first posts:

- why Punk starts core-first
- why modules, adapters, and automation are later

### 3. Manual now, structured now, automated later

Status: active

Primary story:

- `publishing/stories/story-0003-public-memory.md`

Journal sources:
