# GitHub Discussions Community Lab

## Channel / surface

GitHub Discussions.

## Status

Planned/manual.

Nothing has been published yet.

## Purpose

Community Lab raw signal surface for failures, cases, sources, questions, ideas,
and bounded contribution intent.

## Authority

Advisory.

GitHub Discussions replies are raw/advisory signals. They do not create project
truth, roadmap priority, final decisions, acceptance, merge authority, or
canonical topic status.

## Source of truth

Repo artifacts remain source of truth, not discussion replies.

Promotion requires maintainer review or a future gate.

## Publication receipt

A publication receipt is required after manual launch.

Expected receipt path after posting:

```text
publishing/publications/YYYY-MM-DD-community-lab-cycle-0-github-discussions.md
```

Do not create the receipt before the post exists publicly.

## Moderation / privacy boundaries

- Do not collect raw transcript archives.
- Do not ask for secrets, credentials, private logs, customer data, sensitive
  code, or raw prompts.
- Use safe summaries for digests.
- Quote publicly only when the original message says "ok to quote".
- Keep reactions/votes advisory.
- Do not turn chat pressure into roadmap priority.

## Automation boundary

- No bot.
- No integration.
- No GitHub API calls.
- No automatic issue creation.
- No automatic goal creation.
- No automatic PR readiness.
- No Topic Graph runtime.
- No CommunityPunk runtime.
