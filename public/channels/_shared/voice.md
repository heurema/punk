# Punk shared public voice

## Purpose

This file is the human-readable voice note for manual public writing before PubPunk automation exists.

It is the shared style source for all current channels.

## Voice

Write like a builder-researcher, not a marketer.

The voice should feel:

- clear
- pragmatic
- evidence-first
- calm
- honest about scope

## Always do

- state what is active now
- distinguish active work from parked or future work
- use specific dates or phase labels when the timing matters
- explain the tradeoff, not just the claim
- keep paragraphs short
- connect public claims back to Journal, docs, ADRs, or repo artifacts when possible
- mark opinions as opinions

## Never do

- do not imply `punk` is production-ready
- do not imply PubPunk automation already exists in `punk`
- do not imply modules/adapters are current operator paths when they are parked
- do not write hype-first startup copy
- do not hide uncertainty behind confident language
- do not turn the Journal into generic blog filler

## Framing rules

Preferred framing:

- what changed
- what broke
- what survived
- what is active now
- what stays out of scope

## Useful phrases

- experimental, early-stage
- core-first
- local-first
- bounded work
- public build
- project memory
- proof before acceptance

## Avoid or use with care

- autonomous platform
- production-grade
- fully automated
- ready to scale
- revolutionary
- game-changing

## Journal relation

For origin stories, prefer the Journal model:

- what we believed
- what we tried
- what broke
- what survived into Punk
